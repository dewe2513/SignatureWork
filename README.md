## üè• Project 1 - Healthcare Dashboard

**üîó Repository:** [healthcareDashboard](https://github.com/dewe2513/healthcareDashboard)

### Technologies Used
![PowerBI](https://img.shields.io/badge/PowerBI-F2C811?style=flat-square&logo=powerbi&logoColor=black)
![Python](https://img.shields.io/badge/Python-3776AB?style=flat-square&logo=python&logoColor=white)
![Data Analysis](https://img.shields.io/badge/Data-Analysis-orange?style=flat-square)

### Description
This project is a healthcare dashboard I built using PowerBI in my Business Intelligence course last spring. The dashboard takes patient data from various sources and displays it in an interactive PowerBI dashboard. From the dashboard, a user can gain various insights into symptoms, diseases, and common treatments. Creating the dashboard involved data search using Kaggle, pre processing data, building the interactive features, and writing python scripts for data analysis.

The dashboard includes multiple views that let you drill down into specific patient demographics and explore relationships between different medical factors. Users can filter by age groups, symptom severity, and treatment types to discover patterns in the data.

### üí° Why This Matters
This project is meaningful to me due to my longstanding interest in the field of healthcare. It is a representation of how the fields of computer science and healthcare can be merged to create incredibly useful tools and investigate data for the benefit of all. I have a curiosity to explore tons of different fields in my life and this project is a perfect representation of that.

Working on this dashboard showed me how data analytics can actually make a difference in people's lives. Healthcare is going through a huge transformation with technology, and getting to be part of that, even through a class project, was really exciting. It proved to me that I don't have to pick just one field to work in; I can find ways to combine my interests.

## üåê Project 2 - Dynamic Website

**üîó Repository:** [webdev-dynamic](https://github.com/dewe2513/webdev-dynamic)

### Technologies Used
![HTML5](https://img.shields.io/badge/HTML5-E34F26?style=flat-square&logo=html5&logoColor=white)
![CSS3](https://img.shields.io/badge/CSS3-1572B6?style=flat-square&logo=css3&logoColor=white)
![JavaScript](https://img.shields.io/badge/JavaScript-F7DF1E?style=flat-square&logo=javascript&logoColor=black)
![Sustainability](https://img.shields.io/badge/Focus-Sustainability-green?style=flat-square)

### Description
My second project represents the skills I developed in my Web Development course in my final semester at St. Thomas. The project was created and presented in a group of 3 people with each person working on a certain aspect of the project. The project itself is a dynamic web page which populates data from water usage across the world. The website can be used to analyze data and compare and contrast the sustainability efforts of various regions. The course itself was part of a sustainability flag at the University of St. Thomas. This is part of a University wide effort to focus on and be a positive influence on the growing problems of pollution and environmental sustainability representing the overarching ideals of "The Common Good" which drives the mission of St. Thomas.

We built the site using HTML, CSS, and JavaScript to fetch and display real-time data about water consumption patterns. The site features charts and comparison tools that make it easy to see which countries are doing well with water conservation and which ones need improvement. Working in a team taught me a lot about collaboration and how to integrate different people's code into one working project.

### üí° Why This Matters
The project is meaningful to me as it represents the beginning of my exposure to real Web Development. I had limited expereience with Web Development prior to this course. The skills I gained in this class turned out to be invaluable in my job search and my Senior Capstone class. Web Development is a growing industry which contains various different sub fields and the knowledge I gained will continue to prove useful in further years.

Before this class I was kind of intimidated by web development. But once I got into it, I realized how much I enjoyed building things people can actually see and use in their browser. When I was applying for jobs, being able to talk about this project and show something tangible made such a difference. Interviewers could actually visit the site and see what I built. Plus, learning web dev opened up a whole new world of career paths.
